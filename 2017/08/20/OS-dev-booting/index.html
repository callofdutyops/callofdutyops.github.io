发表于 2017-08-20 | 分类于 OS

The Boot Process

We have to realize that when the computer just powered on, there’s little serivices can be used. This situation we called pre-OS environment. At this stage, even a filesystem would be a luxury. So, there must be some simple way to load the OS. The most direct way is to read some bytes on the first sector of disk or some other storage driver like CD-ROM. This sector we called boot sector. But how? Who can do this for us? Luckily, there is a guard man on our computer’s motherboard called BIOS (Basic Input/Output Software), BIOS reads specific sectors of data (in our situation, the first sector, usually 512 bytes in size). But how can BIOS knows the sector is a boot sector? We have to make some special marks on this special sector, this mark called magic number. In this situation, is 0xaa55.

What happened? Why some lines of code can do this magic thing? Let’s look into the boot.bin the nasm produced. We have to use hex editor to display raw data or we’ll get some garbled because normal editor try to interpret the raw data into ASCII.

Look the last two bytes, it’s 0xaa55! Then we look back to our assembly code, dw 0xaa55 do this for us, times just continue padding 0 until it reaches 510 bytes. 510 bytes plused 2 bytes (0xaa55) is exactly 512 bytes. This is our boot sector. About assembly language, you can find a lot of tutorial on Internet.
